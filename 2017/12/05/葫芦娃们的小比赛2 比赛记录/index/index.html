题目大意

在一个黑白图中，有多少个集合，满足集合内所有的元素在同一行或同一列，且颜色是相同的。

思路

记录每一行、每一列的同种颜色的个数，然后直接用组合数计算即可。

B

题目大意

一个游戏：随机选择一个k，胜者加k的平方，败者加k。

问一个游戏结果是否可能存在，多组数据。

思路

一个游戏结果必定是k的三次方的倍数。

二分出x乘y的三次开方，然后看它是否为x和y的因子。

C

此题比赛时没有解出…

题目大意

给你n个数，从中选取恰好k个，使得他们的乘积的末尾0的个数最多，输出最多的0的个数。

思路

考虑动态规划。

记f[i][j][k]表示前i个数，选出j个数，其中有k个因子2的因子5的最大个数。

则转移方程可以写为f[i][j][k] = max(f[i - 1][j][k], f[i - 1][j - 1][k - w2[i]] + w5[i])。

发现空间太大，考虑通过几种方法优化空间：

  • 更改转移方程
    考虑将原来的含义改为f[i][j][k]表示前i个数，选出j个数，其中有k个因子5的因子2的最大个数。
    这样可以将第三维由12800(64×20)优化为5500(27×50)左右，内存约为210MB

  • 使用滚动数组优化第一维。
    事实上，将j的一维倒序循环，即可优化掉第一维（参见01背包的第一维优化方式）。
    那么有转移方程f[j][k] = max(f[j][k], f[j - 1][k - w2[i]] + w5[i])。

反思

  • 没有认真考虑转移方程的设计，没有考虑转移问题

D

题目大意

求树上三点的路径(a->b,a->c)重叠部分的长度。

思路

求出他们两两的最近公共祖先，取深度最大的一个。

然后求出每个点到这个深度最大公共祖先的距离，取最大值即可。

E

此题比赛时没有解出…

题目大意

给你一个数列p[i]，每次操作可以将最后一个数移到数列的最前面。

求∑i=1n∣p[i]−i∣\sum_{i = 1}^{n}|p[i] - i|∑​i=1​n​​∣p[i]−i∣的最小值。

思路

记录有多少个正数，有多少个负数，以及这个数再多少次操作后会改变符号。

每次移动时，所有绝对值号内是正数的对答案贡献-1，是负数的对答案贡献+1。再单独处理最后一个数即可。

s[]记录的是第几次改变有多少个数改变了符号。

  • 没有抓住问题本质，应该多动脑思考
